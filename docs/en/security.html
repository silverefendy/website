Security Documentation

Security controls and production hardening recommendations for authentication, authorization, JWT, passwords, uploads, and OWASP risks.

Authentication

The backend issues short-lived access tokens and refresh tokens. Refresh token hashes are stored server-side.

Authorization

Role middleware enforces admin, seller, and customer access. Frontend protected routes provide user-friendly navigation control.

JWT

Use long random secrets, short access-token expiration, refresh rotation, and secret rotation after incidents.

Password Security

bcrypt protects stored passwords. Add strong password policy, reset flows, and email verification for production.

Upload Security

Validate image type, size, extension, and magic bytes. Re-encode images and scan files before public access.

OWASP Risks

Mitigate broken access control, injection, XSS, CSRF, insecure upload, dependency vulnerabilities, and insufficient logging.
