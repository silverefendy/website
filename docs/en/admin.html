Administrator Guide

Operational guidance for user, role, maintenance, log, monitoring, backup, and restore tasks.

User Management

Administrators review accounts, deactivate suspicious users, verify sellers, and coordinate support escalations.

Role Management

The application uses admin, seller, and customer roles. Future enterprise deployments should add granular permissions and audit logs.

Maintenance

Routine maintenance includes dependency updates, upload cleanup, database optimization, and checking disk usage.

Logs

Use PM2 logs for backend issues, Nginx logs for proxy problems, and MySQL logs for database errors.

Monitoring

Track API health, CPU, memory, disk, database connections, error rates, and order processing failures.

Backup

Back up MySQL and uploads together because order/product records may reference uploaded assets.

Restore

Restore first in staging, validate users/products/orders, then schedule production recovery if needed.
