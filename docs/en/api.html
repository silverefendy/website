API Documentation

Endpoint structure, authentication model, examples, responses, and error handling.

Authentication

Protected endpoints require a bearer access token. Refresh tokens are rotated by the refresh endpoint.

Endpoints

  • /api/auth authentication and profile.
  • /api/products product listing, detail, and reviews.
  • /api/wishlist, /api/cart, and /api/orders customer workflows.
  • /api/seller seller analytics.

Request Examples

GET /api/products?search=phone&page=1&limit=12
POST /api/cart/items { "product_id": 1, "quantity": 2 }
DELETE /api/wishlist/1

Response Examples

{ "success": true, "message": "Products retrieved successfully.", "data": { "products": [], "pagination": { "total": 0 } } }

Error Handling

Common statuses are 401 unauthorized, 403 forbidden, 404 not found, 409 stock conflict, 422 validation error, and 500 server error.
