Panduan Administrator

Panduan operasional untuk manajemen user, role, pemeliharaan, log, monitoring, backup, dan restore.

Manajemen User

Administrator meninjau akun, menonaktifkan user mencurigakan, memverifikasi penjual, dan menangani eskalasi support.

Manajemen Role

Aplikasi memakai role admin, seller, dan customer. Deployment enterprise sebaiknya menambahkan permission granular dan audit log.

Pemeliharaan

Pemeliharaan rutin meliputi update dependency, pembersihan upload, optimasi database, dan pemeriksaan kapasitas disk.

Log

Gunakan log PM2 untuk backend, log Nginx untuk proxy, dan log MySQL untuk error database.

Monitoring

Pantau health API, CPU, memori, disk, koneksi database, error rate, dan kegagalan proses order.

Cadangan Database

Backup MySQL dan direktori upload secara bersamaan karena data order/produk dapat merujuk file upload.

Pemulihan Data

Lakukan restore di staging terlebih dahulu, validasi user/produk/order, lalu jadwalkan pemulihan production jika diperlukan.
