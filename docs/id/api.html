Dokumentasi API

Struktur endpoint, model autentikasi, contoh request, response, dan penanganan error.

Autentikasi

Endpoint terlindungi membutuhkan bearer access token. Refresh token dirotasi oleh endpoint refresh.

Endpoint

  • /api/auth untuk autentikasi dan profil.
  • /api/products untuk daftar, detail, dan ulasan produk.
  • /api/wishlist, /api/cart, dan /api/orders untuk alur pelanggan.
  • /api/seller untuk analitik penjual.

Contoh Request

GET /api/products?search=phone&page=1&limit=12
POST /api/cart/items { "product_id": 1, "quantity": 2 }
DELETE /api/wishlist/1

Contoh Response

{ "success": true, "message": "Products retrieved successfully.", "data": { "products": [], "pagination": { "total": 0 } } }

Penanganan Error

Status umum adalah 401 belum login, 403 akses ditolak, 404 data tidak ditemukan, 409 konflik stok, 422 validasi gagal, dan 500 error server.

Konten yang dicari tidak ditemukan di halaman ini.
