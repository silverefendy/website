Dokumentasi Keamanan

Kontrol keamanan dan rekomendasi hardening untuk autentikasi, otorisasi, JWT, password, upload, dan risiko OWASP.

Autentikasi

Backend menerbitkan access token berdurasi pendek dan refresh token. Hash refresh token disimpan di server.

Otorisasi

Middleware role mengatur akses admin, seller, dan customer. Protected route frontend membantu navigasi sesuai role.

JWT

Gunakan secret panjang dan acak, masa berlaku access token singkat, rotasi refresh token, dan rotasi secret setelah insiden.

Keamanan Password

bcrypt melindungi password tersimpan. Tambahkan kebijakan password kuat, reset password, dan verifikasi email untuk production.

Keamanan Upload

Validasi tipe, ukuran, ekstensi, dan magic bytes gambar. Re-encode gambar dan scan file sebelum akses publik.

Risiko OWASP

Mitigasi broken access control, injection, XSS, CSRF, upload tidak aman, dependency rentan, dan logging yang kurang.
